# HYP-001: Architecture-conditioned error bound

## Claim
Under architecture **A**, formula **X** has a provable or empirically stable error upper bound **Y** (not point accuracy alone).

## Formula / Architecture

**Architecture A (sketch):**
```text
input → module P (precondition) → formula X → module Q (post-check) → output
```

**Formula X (abstract):**
```text
ŷ = f(x; θ)     where f is the core transform under test
```

**Error bound claim:**
```text
|y − ŷ| ≤ Y(ε, n, κ)
  ε  = input perturbation scale
  n  = sample / iteration count
  κ  = architecture conditioning parameter (from module P)
```

## Falsify criteria
- **REJECT** if any run exceeds Y(ε, n, κ) under documented conditions.
- **REJECT** if improving κ does not lower observed max error (bound not architecture-linked).
- **PAUSE** if Y is undefined until user pins domain (what is x, y, f).

## Status
**accepted (toy domain)** via RUN-001 (12/12 PASS)

Toy bound: `Y = C_δ · ε · √n + C_κ · κ` with C_δ=0.3444, C_κ=1.1179 at n=8, ε=1.0.
